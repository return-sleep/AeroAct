# Contributing

Thanks for your interest in contributing to AeroAct.

## How to contribute

1. Fork the repository and create a feature branch from `main`.
2. Keep changes focused and small.
3. Add or update documentation when behavior changes.
4. Run basic checks locally before opening a pull request.
5. Open a pull request with a clear summary and motivation.

## Pull request checklist

- [ ] The change is limited in scope.
- [ ] Documentation is updated (if needed).
- [ ] No private data or secrets are included.
- [ ] Large binaries are not committed directly.

## Reporting issues

Please include:

- Environment details (OS, Python version, CUDA version if relevant)
- Reproduction steps
- Expected behavior and actual behavior
- Minimal logs or traceback
